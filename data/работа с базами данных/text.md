**Учебный материал по теме "Работа с базами данных"**

---

### **1. Введение**
База данных (БД) — это организованное хранилище данных, предназначенное для хранения, управления и извлечения информации. БД используется в различных сферах: от простых приложений до сложных систем управления информацией.

**Зачем нужны базы данных?**
- **Эффективное хранение данных**: Структурированная организация информации.
- **Быстрый доступ к данным**: Индексы и оптимизация запросов.
- **Надежность и целостность данных**: Проверка ограничений (например, уникальность значений).
- **Масштабируемость**: Возможность расширения при увеличении объема информации.

---

### **2. Основные понятия**
**База данных (DB)** — это набор взаимосвязанных данных.  
**Система управления базами данных (СУБД)** — программное обеспечение для работы с БД (например, MySQL, PostgreSQL).  
**Таблица (Table)** — структура данных, состоящая из строк и столбцов.  
**Строка (Row)** — запись данных в таблице.  
**Столбец (Column)** — характеристика данных (например, "Имя", "Дата рождения").  
**Ключ (Key)** — уникальное значение для идентификации строки (первичный ключ, внешний ключ).  

---

### **3. Типы баз данных**
1. **Реляционные БД (RDBMS)**:
   - Данные организованы в таблицы с отношениями между ними.
   - Примеры: MySQL, PostgreSQL, Oracle.
   - Используют язык SQL для работы.

2. **NoSQL-базы данных**:
   - Не требуют строгой структуры (например, документы JSON).
   - Подходят для больших объемов данных и гибкости схемы.
   - Примеры: MongoDB (документная БД), Redis (ключ-значение).

3. **Кубические/мульти维 БД**:
   - Используются для аналитики и обработки больших объемов данных (OLAP).
   - Пример: Oracle OLAP.

4. **Иерархические и сетевые БД**:
   - Редко используются, но подходят для сложных связей (например, деревья, графы).

---

### **4. Язык запросов SQL**
SQL (Structured Query Language) — стандартный язык для работы с реляционными базами данных.

#### Основные команды:
- **SELECT** — извлечение данных.
  ```sql
  SELECT * FROM users; -- выборка всех записей из таблицы "users"
  ```
- **INSERT INTO** — добавление новой строки.
  ```sql
  INSERT INTO users (name, age) VALUES ('Иван', 25);
  ```
- **UPDATE** — изменение данных.
  ```sql
  UPDATE users SET age = 30 WHERE id = 1;
  ```
- **DELETE** — удаление данных.
  ```sql
  DELETE FROM users WHERE id = 1;
  ```

#### Пример запроса:
```sql
SELECT name, age 
FROM employees 
WHERE department = 'IT' 
ORDER BY age DESC;
```
Этот запрос извлекает имена и возраст сотрудников из отдела IT, отсортированных по убыванию возраста.

### **5. Основные операции (CRUD)**
- **Create** — добавление данных (INSERT).
- **Read** — чтение данных (SELECT).
- **Update** — изменение данных (UPDATE).
- **Delete** — удаление данных (DELETE).

### **6. Проектирование базы данных**
1. **Нормализация**:
   - Цель: избежать дублирования и обеспечить целостность.
   - Основные нормы:
     1. Первичная нормальная форма (1NF): данные в виде строк, без повторений.
     2. Вторая нормальная форма (2NF): таблица должна быть в 1NF, и все столбцы зависят от первичного ключа.
     3. Третья нормальная форма (3NF): отсутствие зависимостей между неключевыми полями.

2. **Создание таблиц**:
   ```sql
   CREATE TABLE employees (
       id INT PRIMARY KEY,
       name VARCHAR(100),
       department_id INT,
       FOREIGN KEY (department_id) REFERENCES departments(id)
   );
   ```
   - `PRIMARY KEY` — уникальный идентификатор.
   - `FOREIGN KEY` — связь с другой таблицей.

### **8. Безопасность БД**
1. **Аутентификация**: ограничение доступа к данным по пользователям.
2. **Шифрование**: защита данных от несанкционированного доступа.
3. **Резервное копирование**: регулярное сохранение данных.
4. **Проверка прав** (ACL): контроль над операциями с таблицами.

---

### **9. Примеры задач**
1. **Создать таблицу "Клиенты"**:
   ```sql
   CREATE TABLE clients (
       client_id INT PRIMARY KEY,
       full_name VARCHAR(255),
       email VARCHAR(100) UNIQUE
   );
   ```

2. **Добавить запись**:
   ```sql
   INSERT INTO clients (client_id, full_name, email)
   VALUES (1, 'Иван Иванов', 'ivan@example.com');
   ```

3. **Получить данные по клиенту**:
   ```sql
   SELECT * FROM clients WHERE client_id = 1;
   ```

4. **Обновить почту клиента**:
   ```sql
   UPDATE clients SET email = 'new_email@example.com'
   WHERE client_id = 1;
   ```

5. **Удалить запись**:
   ```sql
   DELETE FROM clients WHERE client_id = 1;
   ```

### **10. Дополнительные темы**
